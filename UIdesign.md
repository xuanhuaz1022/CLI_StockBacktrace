# 股票回测系统 UI 设计方案

## 1. 设计系统

### 1.1 颜色方案

**主色调**：蓝色系（专业、信任）
- 主色：`#1890ff`（Ant Design 蓝色）
- 主色浅：`#e6f7ff`
- 主色深：`#096dd9`

**功能色**：
- 成功色：`#52c41a`
- 警告色：`#faad14`
- 错误色：`#f5222d`
- 信息色：`#1890ff`

**中性色**：
- 文本主色：`#262626`
- 文本次色：`#595959`
- 文本浅色：`#8c8c8c`
- 边框色：`#e8e8e8`
- 背景色：`#f5f5f5`
- 卡片背景：`#ffffff`

### 1.2 排版系统

**字体**：
- 主字体：`-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif`
- 代码字体：`'SFMono-Regular', Consolas, 'Liberation Mono', Menlo, monospace`

**字号系统**：
- 标题 1：24px / 32px
- 标题 2：20px / 28px
- 标题 3：16px / 24px
- 正文：14px / 20px
- 小字：12px / 16px

**字重**：
- 粗体：600
- 常规：400
- 轻量：300

### 1.3 间距系统

**基础间距**：
- 0px: 0
- 4px: 4
- 8px: 8
- 12px: 12
- 16px: 16
- 24px: 24
- 32px: 32
- 48px: 48
- 64px: 64

**布局间距**：
- 容器内边距：16px
- 组件间距：12px
- 卡片间距：16px
- 行间距：8px

### 1.4 阴影系统

**阴影层级**：
- 无阴影：`0 0 0 0 rgba(0, 0, 0, 0)`
- 轻微阴影：`0 2px 8px rgba(0, 0, 0, 0.09)`
- 中等阴影：`0 4px 16px rgba(0, 0, 0, 0.12)`
- 强阴影：`0 8px 32px rgba(0, 0, 0, 0.16)`

### 1.5 圆角系统

**圆角大小**：
- 无圆角：0px
- 小圆角：4px
- 中圆角：8px
- 大圆角：12px
- 圆形：50%

## 2. 页面布局设计

### 2.1 主布局结构

```
┌─────────────────────────────────────────┐
│              Header (顶部导航)            │
├──────────┬──────────────────────────────┤
│          │                              │
│ Sidebar  │      Main Content Area       │
│ (侧边栏) │      (主内容区域)            │
│          │                              │
│          │                              │
└──────────┴──────────────────────────────┘
```

**Header**：
- 高度：64px
- 背景：白色
- 阴影：轻微阴影
- 包含：Logo、导航链接、用户信息、主题切换

**Sidebar**：
- 宽度：200px（展开）/ 80px（折叠）
- 背景：白色
- 边框：右侧 1px 边框
- 包含：导航菜单、折叠按钮

**Main Content Area**：
- 背景：`#f5f5f5`
- 内边距：16px
- 包含：页面内容、面包屑导航

### 2.2 响应式设计

**响应式断点**：
- 移动端：< 768px
- 平板：768px - 1024px
- 桌面：> 1024px

**适配策略**：
- 移动端：
  - 隐藏侧边栏，使用底部导航或顶部汉堡菜单
  - 单列布局
  - 简化图表展示
- 平板：
  - 侧边栏可折叠
  - 双列布局
  - 适当缩小图表
- 桌面：
  - 完整布局
  - 多列布局
  - 完整图表展示

### 2.3 导航结构

**顶部导航**：
- Logo
- 主要功能区链接
- 用户头像/信息
- 主题切换按钮
- 通知图标

**侧边导航**：
- 仪表盘
- 数据管理
  - 上传数据
  - 数据列表
- 策略管理
  - 创建策略
  - 策略列表
- 回测管理
  - 新建回测
  - 回测列表
- 结果分析
- 系统设置

**面包屑导航**：
- 显示当前页面路径
- 提供快速导航

## 4. 组件设计

### 4.1 基础组件

**按钮**：
- 主要按钮：主色背景，白色文字
- 次要按钮：白色背景，主色边框
- 文本按钮：无背景，主色文字
- 危险按钮：红色背景，白色文字

**表单**：
- 输入框：白色背景，灰色边框
- 下拉选择：白色背景，灰色边框
- 日期选择器：白色背景，灰色边框
- 开关：主色开关条，白色开关按钮

**表格**：
- 表头：浅灰色背景，深色文字
- 行：白色背景，交替行浅灰色背景
- 边框：1px 灰色边框
- 排序：点击表头排序

**卡片**：
- 白色背景
- 轻微阴影
- 16px 内边距
- 8px 圆角

### 4.2 布局组件

**栅格**：
- 24 列栅格系统
- 响应式断点支持
- 间距可配置

**布局**：
- 容器：固定宽度或流体宽度
- 分割线：1px 灰色线条
- 空间：可配置高度的空白区域

### 4.3 图表组件

**折线图**：
- 用于展示时间序列数据
- 支持多数据系列
- 支持交互和缩放

**柱状图**：
- 用于比较不同类别的数据
- 支持堆叠和分组模式

**饼图**：
- 用于展示比例数据
- 支持环形饼图

**K线图**：
- 用于展示股票价格数据
- 支持技术指标叠加

### 4.4 功能组件

**模态框**：
- 居中显示
- 半透明背景遮罩
- 可拖动和调整大小

**抽屉**：
- 从右侧滑出
- 可调整宽度
- 支持多层嵌套

**通知**：
- 顶部弹出
- 自动消失
- 支持不同类型（成功、警告、错误、信息）

**加载**：
- 页面加载：骨架屏
- 组件加载：旋转动画
- 按钮加载：按钮内旋转动画

## 5. 交互设计

### 5.1 加载状态

**页面加载**：
- 骨架屏：显示页面结构的占位符
- 加载动画：居中显示旋转动画

**组件加载**：
- 按钮加载：按钮内显示旋转动画，禁用按钮
- 表格加载：表格区域显示加载动画
- 图表加载：图表区域显示加载动画

**数据加载**：
- 无限滚动：滚动到底部自动加载更多数据
- 分页加载：点击分页按钮加载数据

### 5.2 错误处理

**全局错误**：
- 顶部通知条：显示全局错误信息
- 错误页面：当页面加载失败时显示

**表单错误**：
- 实时验证：输入时显示错误提示
- 提交验证：提交时显示错误提示
- 错误样式：红色边框和错误图标

**网络错误**：
- 重试按钮：网络错误时显示重试按钮
- 离线提示：网络断开时显示离线提示

### 5.3 反馈机制

**操作成功**：
- 顶部通知：显示操作成功信息
- 成功图标：操作成功时显示成功图标

**操作确认**：
- 确认对话框：危险操作前显示确认对话框
- 二次确认：重要操作需要二次确认

**进度反馈**：
- 进度条：显示操作进度
- 百分比：显示完成百分比
- 状态提示：显示当前操作状态

**hover 反馈**：
- 按钮：hover 时轻微变暗
- 链接：hover 时显示下划线
- 卡片：hover 时轻微上浮和阴影增强

## 6. 响应式设计

### 6.1 移动端适配

**布局调整**：
- 顶部导航：简化为 Logo、汉堡菜单和用户头像

- 内容区域：单列布局，减少内边距

**组件调整**：
- 按钮：增大点击区域
- 表单：增大输入框
- 表格：水平滚动或简化显示
- 图表：简化展示，减少数据点

**交互优化**：
- 触摸友好：增大触摸目标

- 底部导航：添加底部导航栏

**布局调整**：
- 侧边栏：可折叠，默认展开
- 内容区域：双列布局
- 间距：适当减小间距

**组件调整**：
- 表格：可水平滚动
- 图表：适当缩小
- 卡片：调整大小

**交互优化**：
- 触摸和鼠标混合支持
- 侧边栏手势滑动

### 6.3 桌面端适配

**布局调整**：
- 完整三栏布局
- 充分利用屏幕空间
- 多列布局

**组件调整**：
- 表格：完整显示所有列
- 图表：完整展示，支持交互
- 卡片：标准大小

**交互优化**：
- 鼠标悬停效果
- 键盘快捷键
- 拖拽功能

## 8. 实现建议

### 8.1 技术栈选择

- **前端框架**：React + TypeScript
- **UI 库**：Ant Design
- **图表库**：ECharts
- **状态管理**：Redux Toolkit
- **路由**：React Router
- **构建工具**：Vite

### 8.2 性能优化

- **代码分割**：路由级别的懒加载
- **Tree Shaking**：移除未使用的代码
- **虚拟滚动**：长列表优化
- **缓存策略**：合理使用缓存
- **图片优化**：使用适当的图片格式和大小

### 8.3 可访问性

- **WCAG 2.1 标准**：遵循可访问性标准
- **键盘导航**：支持键盘操作
- **屏幕阅读器**：支持屏幕阅读器
- **颜色对比度**：确保足够的颜色对比度
- **ARIA 标签**：适当使用 ARIA 标签

### 8.4 国际化

- **多语言支持**：使用 i18next
- **日期和时间格式化**：使用国际化日期时间格式
- **数字格式化**：使用国际化数字格式
- **文本方向**：支持从右到左的文本方向

## 9. 总结

本 UI 设计方案基于前端架构设计文档，提供了一套完整的、现代化的股票回测系统 UI 设计。设计遵循了一致性、响应式、可访问性、性能和易用性的原则，为用户提供了专业、高效、易用的回测工具。

设计方案包括了完整的设计系统、页面布局、核心页面设计、组件设计、交互设计、响应式设计和主题系统，为开发团队提供了详细的设计规范和实现指南。

通过这套设计方案，股票回测系统将能够提供优秀的用户体验，帮助用户更高效地进行策略回测和分析，从而做出更明智的投资决策。
