# 股票交易回测系统

一个高性能的多线程C++股票交易回测系统框架，支持多种策略的并行回测和性能评估。

## 系统架构

### 核心模块

1. **数据处理模块** (`data_handler.h/cpp`)
   - 负责数据加载、预处理和访问
   - 支持从CSV文件加载历史行情数据
   - 提供数据查询接口

2. **策略模块** (`strategy.h/cpp`)
   - 策略基类，定义策略接口
   - 支持自定义交易策略
   - 示例策略：移动平均线策略

3. **回测引擎** (`backtest_engine.h/cpp`)
   - 协调数据、策略和执行回测
   - 支持多线程并行回测
   - 管理回测结果

4. **线程池** (`thread_pool.h/cpp`)
   - 实现多线程任务调度
   - 支持并行回测多个策略

5. **性能评估** (`performance.h/cpp`)
   - 计算回测结果的性能指标
   - 包括收益率、最大回撤、夏普比率、索提诺比率等

6. **配置管理** (`config.h/cpp`)
   - 加载和管理系统配置
   - 支持从配置文件读取参数

7. **日志模块** (`logging.h/cpp`)
   - 系统日志记录
   - 支持不同级别的日志输出

## 系统特性

- **高性能**：使用多线程并行处理，提高回测速度
- **可扩展**：支持自定义策略和数据格式
- **完整的性能评估**：提供多种性能指标
- **配置灵活**：通过配置文件管理系统参数
- **日志记录**：详细的系统运行日志

## 构建与运行

### 构建步骤

1. 确保安装了CMake和C++17编译器
2. 创建构建目录：
   ```bash
   mkdir build
   cd build
   ```
3. 运行CMake：
   ```bash
   cmake ..
   ```
4. 编译项目：
   ```bash
   make
   ```

### 运行步骤

1. 准备历史行情数据，格式为CSV文件，包含以下字段：
   - 日期（YYYY-MM-DD）
   - 开盘价
   - 最高价
   - 最低价
   - 收盘价
   - 成交量

2. 修改`config.ini`配置文件，设置数据路径和其他参数

3. 在`main.cpp`中取消注释数据加载代码，指定数据文件路径

4. 运行回测：
   ```bash
   ./stock_backtrack
   ```

## 自定义策略

要创建自定义策略，只需继承`Strategy`类并实现以下方法：

- `on_bar(const std::string& symbol, const Bar& bar)`：处理每个时间周期的数据
- `get_name() const`：返回策略名称

例如：

```cpp
class MyStrategy : public Strategy {
public:
    void on_bar(const std::string& symbol, const Bar& bar) override {
        // 实现交易逻辑
    }
    
    std::string get_name() const override {
        return "MyStrategy";
    }
};
```

## 性能指标说明

- **收益率**：回测期间的总收益率
- **最大回撤**：回测期间的最大亏损百分比
- **夏普比率**：风险调整收益率，衡量单位风险的超额收益
- **索提诺比率**：类似夏普比率，但只考虑下行风险

## 示例数据

系统支持CSV格式的历史行情数据，示例格式：

```csv
date,open,high,low,close,volume
2023-01-01,100.0,105.0,98.0,102.0,1000000
2023-01-02,102.0,108.0,100.0,106.0,1200000
...
```

## 注意事项

- 系统默认使用4个线程进行并行回测，可以根据硬件配置调整
- 数据文件需要按照指定格式准备，否则可能导致加载失败
- 策略实现需要注意线程安全性，避免并发访问冲突
- 回测结果仅供参考，实际交易可能受到市场流动性、滑点等因素影响
